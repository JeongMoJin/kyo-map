# 공가지도 최종 발표/데모 런북

작성일: 2026-06-21
목표: 네트워크/브라우저/질문 변수에도 흔들리지 않는 최종 발표 흐름
최종 업데이트: 2026-06-21 실서비스 전환 구현 반영

## 0. 실서비스 전환 후 데모 핵심 흐름

1. `/` 홈 지도에서 상단 티커와 `Production data` 패널을 먼저 보여준다.
2. "현재 서버가 `data.go.kr` 영주시 공개 빈집 현황 CSV를 직접 읽어 950건, 기준일 2025-07-10을 확인했습니다"라고 말한다.
3. `/dashboard`로 이동해 공공데이터 패널의 우선 조사 후보군과 키 상태를 보여준다.
4. `/house/H00001` 상세로 이동해 `Real AI analysis`의 `AI 분석 실행` 버튼을 누른다.
5. 현재 환경은 `OPENAI_API_KEY`가 없어 `local_policy_scoring` fallback이 뜨는 것을 보여주고, 발표 전 키 설정 시 OpenAI Responses API 결과가 같은 위치에 표시된다고 설명한다.

발표 전 최우선 준비:

- `OPENAI_API_KEY`를 설정한 실제 OpenAI 분석 성공 캡처 1장
- `DATA_GO_KR_SERVICE_KEY`를 설정한 ODCloud API 성공 캡처 1장
- 키가 없을 때 fallback이 정상 표시되는 캡처 1장
- 영주시 공개 CSV 원천 URL 캡처 1장
- `/api/public-data/yeongju?limit=0` JSON 응답 캡처 1장

## 2. 발표 시작 30초 안에 보여줄 핵심 문제

"빈집 문제는 이미 등록된 빈집을 보는 것만으로는 늦습니다. 지자체는 현장조사 전에 어떤 주소를 먼저 확인해야 할지 판단해야 합니다. 공가지도는 공공데이터와 AI 분석 흐름을 이용해 미등록 빈집 후보를 사전에 좁히고, 위험도와 우선조사 순서를 제안하는 시제품입니다."

주의: 첫 30초 안에 "확정 판정이 아니라 후보 탐지"를 반드시 말한다.

## 3. 첫 번째 "와" 포인트

지도 첫 화면에서 `전국 공가 100건`과 색상별 후보 마커가 보이는 순간. 단, 말은 이렇게 해야 한다.

"현재 화면은 샘플 후보 100건으로 구현한 데모입니다. 실제 실증에서는 건축물대장, 가명 전력사용 패턴, 위성영상, 안심구역, 교통 접근성 데이터를 연결해 같은 흐름으로 후보를 산출합니다."

## 4. 3분 발표 버전 데모 흐름

1. `/` 지도 첫 화면: "미등록 빈집 후보를 지도에서 한 번에 본다."
2. 필터 사이드바: "귀촌/창업/철거, 신뢰도, 안심구역으로 조사 대상을 좁힌다."
3. 후보 하나 클릭 또는 `/house/H00018`: "상세에서 AI 추정 근거와 전력사용량, 행정 우선순위를 본다."
4. `/dashboard`: "지자체 담당자는 Top10과 CSV로 우선조사 목록을 뽑는다."
5. 마무리: "공가지도는 현장조사를 대체하지 않고, 현장조사를 더 정확하게 준비하게 한다."

## 5. 5분 발표 버전 데모 흐름

1. 문제 제기 40초: 미거주 주택 160만 호, 법령상 빈집 약 13만 호의 간극.
2. `/` 지도 60초: 후보 지도, 필터, 마커 설명.
3. `/house/H00018` 90초: 철거/위험 후보 상세. 전력사용량, 안심구역, 우선순위 99점, 담당 부서/48시간 점검.
4. `/house/H00001` 45초: 귀촌/재생 후보 상세. "철거만이 아니라 활용도 판단한다."
5. `/dashboard` 60초: Top10, 지역 분포, CSV, 공식 근거 패널.
6. 검증 계획 30초: 빈집대장 매칭, 현장확인, 점수 보정.
7. 차별성 15초: 기존 플랫폼은 등록정보, 공가지도는 사전 후보.

## 6. 7분 발표 버전 데모 흐름

1. 오프닝 45초: 문제 규모와 행정 간극.
2. 기존 방식 한계 45초: 등록/확인된 빈집 중심 관리의 한계.
3. 솔루션 정의 45초: 공공데이터 기반 후보 탐지, 확정 판정 아님.
4. 지도 데모 90초: 후보 분포, 필터, 마커 팝업, 상세 이동.
5. 상세 데모 100초: AI 분석 탭, 전력사용량 탭, 우선순위 산식, 행정 조치.
6. 대시보드 데모 90초: Official evidence, Top10, CSV, 지역/용도 차트.
7. 데이터/AI 증거 45초: 현재 MVP와 실증 연동 예정 구분.
8. 경쟁 차별성 30초: 빈집애와 역할 차이.
9. 확장/마무리 30초: 지자체 PoC -> SaaS -> 유휴공간 확장.

## 9. 장애 대응 플랜

### 인터넷 장애 시

1. 로컬 `next start` 서버로 전환.
2. 지도 타일이 안 뜨면 백업 스크린샷 PDF로 전환.
3. "실제 서비스는 지도 위 후보와 상세 근거가 핵심이므로, 백업 화면으로 흐름을 설명드리겠습니다"라고 말한다.

### API/외부 자산 장애 시

현재 앱은 자체 API가 없지만 외부 지도 타일과 `picsum.photos` 이미지가 있다.

1. 지도 타일 장애: 지도 캡처 이미지로 대체.
2. 상세 이미지 장애: 상세 화면 아래 분석 카드/전력차트를 중심으로 설명.
3. 배포 장애: 로컬 서버 또는 녹화 영상.

### 데모 실패 시 말로 전환하는 플랜

1. "데모가 잠시 불안정하니 동일 흐름을 캡처 화면으로 설명드리겠습니다."
2. 슬라이드 순서: 지도 -> 상세 AI 분석 -> 전력 차트 -> 대시보드 Top10 -> CSV.
3. 마지막에 "로컬 빌드와 배포 빌드는 발표 전 통과했고, 실패 지점은 네트워크/브라우저 이슈"라고 짧게 정리.

3. `/house/H00018`: 철거/위험 후보, 우선순위 99점, 행정 조치.
4. `/house/H00001`: 귀촌/재생 후보, 전력사용량 낮음, 재생 시나리오.

## 11. 반드시 준비해야 할 30-60초 백업 영상 목록

1. 지도에서 후보 클릭 후 상세 이동.
2. 상세 `AI 분석` 탭 설명 후 `전력사용량` 탭 전환.
3. 대시보드 Top10 확인 후 CSV 다운로드.
4. 모바일에서 필터 바텀시트 열고 기준 변경.
5. 발표 전체 압축 영상: 지도 10초 -> 상세 20초 -> 대시보드 20초.

## 12. 발표자가 외워야 할 방어 문장

- "공가지도는 빈집 확정 판정 시스템이 아니라 현장조사 전 후보 추천 시스템입니다."
- "현재 시제품은 샘플 데이터 기반이며, 실증 단계에서 지자체 빈집대장과 현장확인 결과로 검증합니다."
- "AI라는 이름으로 결론만 내는 것이 아니라, 후보 점수와 행정 우선순위의 근거를 화면에 보여주는 것이 핵심입니다."
- "기존 플랫폼과 경쟁하기보다, 기존 빈집관리 체계의 앞단을 보완합니다."
- "오탐 가능성은 전제로 두고, 지자체 검증 결과를 다시 점수 기준에 반영하는 구조로 발전시키겠습니다."
