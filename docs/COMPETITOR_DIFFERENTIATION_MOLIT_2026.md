# 공가지도 경쟁 서비스 대비 차별성 감사

작성일: 2026-06-21
외부 확인 출처: 빈집애, 한국부동산원 빈집정보시스템, 국토연구원 국토정책Brief
최종 업데이트: 2026-06-21 실서비스 전환 구현 반영

## 0. 실서비스 전환 후 차별성 보강

공가지도는 이제 "화면만 있는 후보 서비스"가 아니라 실제 공개 빈집 현황 CSV를 서버에서 읽어 리 단위 후보군과 우선 조사 점수를 만든다. 빈집애가 등록/공식 확인된 빈집 정보를 중심으로 보여주는 플랫폼이라면, 공가지도는 공개 데이터와 AI 분석을 이용해 행정조사 전 후보군을 좁히는 앞단 의사결정 보조 서비스로 포지셔닝할 수 있다.

심사위원에게 말할 수 있는 업데이트 문장:

1. "빈집애가 등록된 빈집을 보여준다면, 공가지도는 공공데이터를 직접 읽어 현장조사 전 후보군을 우선순위로 좁힙니다."
2. "현재 서버는 영주시 공개 빈집 현황 950건을 직접 수집하고, 리 단위 위험 후보군을 계산합니다."
3. "정확 지번과 소유자는 공개하지 않고, 공개 화면은 리 단위 후보군으로 제한해 개인정보 위험을 낮췄습니다."
4. "OpenAI Responses API는 후보 1건의 조치안, 체크리스트, 데이터 한계를 구조화 JSON으로 생성하도록 붙였습니다."
5. "API 키나 네트워크가 실패해도 로컬 정책 산식 fallback으로 데모와 내부 검토가 멈추지 않습니다."

## 1. 핵심 판정

공가지도의 차별성은 "등록된 빈집 조회"가 아니라 "미등록 빈집 후보의 사전 발굴과 행정조사 우선순위 추천"에 있다. 이 방향은 충분히 설득력 있다. 다만 현재 화면 일부는 기존 공식 플랫폼보다 더 큰 범위의 실제 운영 서비스를 이미 하는 것처럼 보일 수 있으므로, 발표에서는 기존 서비스와 경쟁/대체가 아니라 "실태조사 앞단 보완"으로 말해야 한다.

## 2. 기존 서비스가 하는 일

| 서비스/체계 | 확인한 역할 | 공가지도와의 관계 |
| --- | --- | --- |
| 빈집애 | 전국 빈집정보, 빈집 현황, 빈집정비사업 안내, 활용 사례, 소유자 공개 동의 매물 정보 제공 | 등록/공개/정비 현황 중심 공식 플랫폼 |
| 한국부동산원 빈집정보시스템 | 빈집의 효율적 정비를 위한 빈집추정 및 현황관리 정보시스템 | 행정 내부/공식 현황관리 체계 |
| 범부처/한국부동산원 향후 분석 서비스 | 빈집 현황, 실태조사 결과, 행정통계, 외부데이터 결합을 통한 발생·확산 예측 및 입지 분석 목표 | 공가지도와 방향이 유사해질 수 있는 미래 경쟁/협력 영역 |

## 3. 공가지도가 다르게 해야 하는 일

| 비교 관점 | 기존 플랫폼 | 공가지도 현재 구현 | 차별성의 강도 |
| --- | --- | --- | --- |
| 핵심 대상 | 이미 확인/등록/공개된 빈집 | 미등록 또는 행정 확인 전 후보 | 강함 |
| 사용자 | 일반 국민, 빈집 소유자/수요자, 행정 담당자 | 지자체 조사/정비 담당자 중심 | 중간-강함 |
| 화면 | 현황, 정비사업, 활용 사례, 매물 | 지도, 후보 필터, 상세 분석, 대시보드 Top10 | 강함 |
| 데이터 | 공식 현황/실태조사 중심 | 샘플 데이터로 공공데이터 융합 구조 구현 | 현재는 약함 |
| AI | 기존/향후 시스템도 예측 분석 방향 있음 | ViT/LSTM/GPT-4o 개념 파이프라인 + 산식 스코어링 | 현재 증거 약함 |
| 행정 가치 | 현황 관리/정보 제공 | 현장조사 전 우선순위/담당부서/조치 추천 | 강함 |
| 검증 루프 | 공식 조사 절차 | 검증 계획은 문서화, 구현은 미흡 | 보완 필요 |

## 4. 심사위원 관점 리스크

### P0: 빈집애도 향후 발생·확산 예측/입지 분석을 목표로 한다

- 외부 자료상 범부처/한국부동산원은 빈집 현황, 실태조사 결과, 행정통계, 외부 데이터 결합 기반 분석 서비스를 개발하려는 방향을 갖고 있다.
- 따라서 "우리만 AI 분석을 한다"가 아니라 "공가지도는 최종 발표 시점에 실제 웹 시제품으로 사전 후보 탐지와 행정 우선순위 화면을 보여준다"로 차별화해야 한다.

### P1: 공가지도 화면의 "전국/실시간/AI 엔진" 표현은 기존 공식 플랫폼보다 과장돼 보일 수 있다

- 경쟁 서비스와 비교할수록 신뢰성이 중요하다.
- 공식 플랫폼을 낮춰 말하지 말고 "역할이 다르다"로 정리한다.

### P1: 오탐/검증/피드백 구조가 UI에 약하다

- 미등록 후보 탐지 서비스는 오탐이 당연히 존재한다.
- 상세 화면 하단 또는 발표자료에서 `후보 탐지 -> 지자체 빈집대장 매칭 -> 현장 확인 -> 모델 보정` 루프를 반드시 보여줘야 한다.

## 5. 지자체 도입 이유

| 지자체 업무 문제 | 공가지도 제안 가치 | 현재 구현 근거 |
| --- | --- | --- |
| 모든 후보를 같은 강도로 조사하기 어렵다 | 우선조사 Top10과 현장확인 필요 점수 | `/dashboard`, `lib/priority.ts` |
| 빈집 후보 근거가 흩어져 있다 | 전력, 건물, 입지, 위험도 근거를 상세 화면에 모음 | `/house/[id]` |
| 빈집을 찾아도 다음 조치가 어렵다 | 담당 부서, 조치 기한, 철거/귀촌/창업 시나리오 | `components/HouseDetailView.tsx` |
| 정책 활용까지 연결하기 어렵다 | 재생 적합도와 추천 용도 표시 | 상세/대시보드 |
| 보고서/행정 공유가 필요하다 | CSV 내보내기, 월간 리포트 인쇄 버튼 | `/dashboard` |

## 6. 심사위원에게 말할 한 문장 5개

1. "빈집애가 공개·등록된 빈집 정보를 한곳에서 보여준다면, 공가지도는 행정 확인 전 단계의 빈집 후보를 먼저 좁혀 줍니다."
2. "공가지도는 빈집을 확정 판정하지 않고, 지자체가 현장조사 전에 어디를 먼저 볼지 판단하는 우선조사 추천 도구입니다."
3. "지도에서 후보를 찾는 데서 끝내지 않고, 상세 화면에서 전력사용 패턴과 위험도, 담당 부서, 권장 조치까지 연결했습니다."
4. "기존 공식 시스템을 대체하려는 것이 아니라, 실태조사와 빈집대장 정비의 앞단을 보완하는 시제품입니다."
5. "현재는 샘플 데이터 기반 MVP이며, 실증 단계에서는 지자체 빈집대장과 현장 확인 결과로 후보 점수와 우선순위를 검증하겠습니다."

## 7. 발표 비교표 권장안

| 구분 | 기존 빈집 플랫폼 | 공가지도 |
| --- | --- | --- |
| 핵심 질문 | 이미 확인된 빈집이 어디 있나? | 아직 확인되지 않은 후보 중 어디를 먼저 조사해야 하나? |
| 데이터 단계 | 등록/실태조사 후 정보 | 조사 전 후보 데이터 |
| 사용자 행동 | 조회, 정보 확인, 매물/사례 확인 | 후보 필터링, 상세 근거 확인, 우선조사 결정 |
| 결과물 | 현황/매물/정비사업 정보 | 우선순위, 위험도, 담당부서, 정책 활용 시나리오 |
| 안전한 표현 | 공식 플랫폼 | 사전 스크리닝 보조 도구 |
