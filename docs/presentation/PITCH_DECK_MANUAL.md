# 공가지도 발표자료 직접 제작 가이드

이 문서는 PowerPoint 또는 미리캔버스에서 사람이 직접 발표자료를 만들기 위한 제작 지침이다. PPTX, 웹 슬라이드, PNG 슬라이드 자동 생성은 사용하지 않는다.

## 전체 톤

- 담백한 공모전 발표자료.
- 흰 배경 중심.
- 네이비 제목, 블루/그린 포인트.
- Pretendard 우선, 없으면 Noto Sans KR.
- 본문 글자 14pt 이상.
- 과한 장식, AI 느낌의 보라색 그라데이션, 복잡한 애니메이션 금지.
- 일반적인 미리캔버스 IR 템플릿을 고르되, 공공기관 제안서처럼 정돈된 레이아웃으로 수정.
- 핵심 메시지는 `빈집 확정 판정`이 아니라 `현장조사 전 사전 스크리닝`과 `우선조사 추천`.

## 공통 디자인 규칙

### 색상

- 제목: `#0F2A44`
- 강조 제목 또는 CTA: `#2F80ED`
- 긍정/활용 가능성 포인트: `#2EAD73`
- 경고/위험도 포인트: `#F2994A` 또는 `#EB5757`
- 본문: `#4B5563`
- 보조 설명: `#7A8A9A`
- 배경: `#FFFFFF` 또는 `#F7FAFC`
- 표/카드 테두리: `#D8E5F2`

### 타이포그래피

- 표지 제목: 44-56pt.
- 본문 슬라이드 제목: 26-34pt.
- 본문: 14-18pt.
- 캡션/출처: 9-11pt.
- 한글 줄바꿈은 어절 단위로 자연스럽게 조정.
- 한 줄에 문장을 너무 길게 넣지 않는다.

### 표 스타일

- 헤더 행은 네이비 배경 또는 아주 연한 블루 배경.
- 선은 얇게.
- 셀 안 여백은 넉넉하게.
- 비교표는 3열 이하로 유지.
- 공가지도 열은 블루 또는 그린으로 은은하게 강조.

### 카드 스타일

- 둥근 모서리 8-12px.
- 흰 배경.
- 얇은 테두리.
- 그림자는 거의 쓰지 않거나 아주 약하게.
- 카드 하나에는 메시지 하나만.

### 이미지 사용 규칙

- 실제 서비스 화면 캡처를 우선 사용.
- 지도/대시보드/후보 상세 패널은 흐리게 처리하지 않는다.
- 기사 이미지는 제목과 출처가 보이게 작게 캡처.
- 지도 화면에 개인정보가 있으면 가린다.
- 출처가 필요한 이미지는 하단에 작은 글씨로 출처와 날짜를 쓴다.

### 기사/출처 표기 규칙

- 통계 카드 하단: `출처: 기관명, 자료명, 날짜`.
- 기사 카드 하단: `언론사, 기사일`.
- 출처 없는 숫자는 쓰지 않는다.
- 검증 전 수치는 `예시`, `샘플`, `실증 예정`으로 표시한다.

### 줄바꿈 규칙

- 제목은 1-2줄.
- 본문은 2-4개 bullet 이내.
- 슬라이드 한 장에는 핵심 문장 하나를 크게 둔다.
- 금지: 작은 글씨로 긴 설명을 가득 채우기.

## 페이지별 상세 가이드

## 1페이지. 표지

1. 페이지 제목
   - `공가지도`
   - 부제: `공공데이터와 AI로 미등록 빈집 후보를 찾아내는 도시 데이터 플랫폼`

2. 이 페이지의 목적
   - 서비스 이름과 핵심 정의를 첫 화면에서 각인한다.
   - 심사위원이 바로 “구현된 지도 기반 서비스”라고 이해하게 한다.

3. 넣을 문장
   - `공공데이터와 AI로 미등록 빈집 후보를 찾아내는 도시 데이터 플랫폼`
   - 하단 작은 정보: 이름, 대회명, 서비스 URL.

4. 넣을 이미지/자료
   - 로고 또는 `공가지도` 워드마크.
   - 실제 배포 사이트 첫 화면 캡처.
   - 이름/대회명.

5. 자료를 어디서 구해야 하는지
   - 사이트 캡처: 로컬 또는 배포 URL에서 `/` 또는 `/map` 화면 캡처.
   - 로고: 별도 로고가 없으면 텍스트 워드마크로 제작.

6. 배치 방법
   - 권장: 왼쪽 40%에 제목과 부제, 오른쪽 60%에 서비스 화면.
   - 대안: 상단 제목, 하단 전체 너비 서비스 화면.
   - 화면 캡처는 테두리 1px, 모서리 10px.

7. 색상/강조 방법
   - 제목 네이비.
   - `AI`, `미등록 빈집 후보`만 블루 강조.
   - 배경은 흰색.

8. 발표 멘트
   - `공가지도는 등록된 빈집을 조회하는 지도를 넘어, 공공데이터와 AI로 아직 등록되지 않은 빈집 후보를 먼저 찾아내는 도시 데이터 플랫폼입니다.`

9. 주의할 점
   - `빈집 확정`, `정확도` 같은 표현을 넣지 않는다.
   - 서비스 화면이 작게 보이지 않게 한다.

10. 심사기준 연결
   - 공공데이터 활용성.
   - 실제 구현 여부.
   - 서비스 확장 가능성.

## 2페이지. 문제 제기

1. 페이지 제목
   - `빈집 문제는 늘어나지만, 발견과 대응은 늦습니다`

2. 이 페이지의 목적
   - 왜 이 문제가 공공데이터로 풀어야 하는 도시 문제인지 설명한다.

3. 넣을 문장
   - `빈집은 단순히 비어 있는 주택이 아니라 지역 안전, 도시 미관, 생활 만족도, 행정 조사비용과 연결되는 도시 관리 문제입니다.`
   - `통계상 미거주 주택과 행정상 관리되는 빈집 사이에는 큰 간극이 있습니다.`

4. 넣을 이미지/자료
   - 빈집 증가 통계 1개.
   - 빈집 관련 기사 2-3개.
   - 출처와 날짜.

5. 자료를 어디서 구해야 하는지
   - 대한민국 정책브리핑: 2024년 인구주택총조사 결과.
   - 국토연구원 빈집 관련 브리프.
   - 네이버뉴스 또는 언론사 사이트에서 `빈집 증가`, `빈집 안전`, `빈집 정비` 검색.

6. 배치 방법
   - 왼쪽: 큰 통계 카드 1개.
   - 오른쪽: 기사 헤드라인 카드 2-3개.
   - 하단: 문제 영향을 4개 키워드로 정리.

7. 색상/강조 방법
   - 통계 숫자는 네이비 또는 블루.
   - 위험/방치 관련 키워드는 오렌지로만 약하게 강조.

8. 발표 멘트
   - `빈집은 단순히 비어 있는 집의 문제가 아니라 지역 안전, 도시 미관, 생활 만족도와 연결되는 도시 관리 문제입니다. 특히 통계상 미거주 주택과 행정적으로 관리되는 빈집 사이에는 간극이 있어서, 어디를 먼저 확인해야 하는지가 중요합니다.`

9. 주의할 점
   - 통계 숫자는 반드시 출처를 함께 둔다.
   - 기사를 공포스럽게 꾸미지 않는다.

10. 심사기준 연결
   - 문제 중요성.
   - 공공데이터 기반 해결 필요성.

## 3페이지. 기존 방식의 한계

1. 페이지 제목
   - `현재 방식은 확인된 빈집 중심입니다`

2. 이 페이지의 목적
   - 기존 플랫폼이나 관리 방식이 틀렸다는 말이 아니라, 역할이 다르다는 점을 설명한다.

3. 넣을 문장
   - 카드 1: `발견이 늦다`
   - 카드 2: `조사 우선순위가 어렵다`
   - 카드 3: `활용 연결이 약하다`
   - 보조문: `등록되거나 확인된 빈집 중심의 관리는 현장조사 전 의사결정에는 한계가 있습니다.`

4. 넣을 이미지/자료
   - 기존 빈집 플랫폼 참고 화면 1개.
   - 행정 조사 프로세스 간단 도식.

5. 자료를 어디서 구해야 하는지
   - 기존 공공 빈집 플랫폼 화면은 직접 접속 후 캡처.
   - 법/제도 설명은 국토부, 지자체, 국토연구원 자료 참고.

6. 배치 방법
   - 상단: 기존 방식 한 문장 요약.
   - 중앙: 3개 카드.
   - 하단: `확인된 정보 관리`와 `사전 후보 탐지` 사이의 빈칸을 표시.

7. 색상/강조 방법
   - 카드 제목은 네이비.
   - 한계 키워드는 오렌지 아이콘 또는 선으로 표시.

8. 발표 멘트
   - `현재 방식은 이미 등록되거나 확인된 빈집을 중심으로 관리되기 때문에, 현장조사 전에 어디를 먼저 확인해야 하는지 판단하기 어렵습니다. 공가지도는 이 앞단의 사전 스크리닝을 돕는 서비스입니다.`

9. 주의할 점
   - 기존 서비스를 깎아내리지 않는다.
   - `기존 플랫폼은 필요 없고 공가지도만 필요하다`는 뉘앙스를 피한다.

10. 심사기준 연결
   - 기존 방식 대비 차별성.
   - 행정 현장 적용성.

## 4페이지. 핵심 인사이트

1. 페이지 제목
   - `필요한 것은 조회가 아니라 사전 탐지입니다`

2. 이 페이지의 목적
   - 서비스의 가장 중요한 관점을 한 문장으로 전환한다.

3. 넣을 문장
   - `공가지도의 핵심은 빈집을 확정 판정하는 것이 아니라, 현장조사 전에 확인해야 할 빈집 후보를 먼저 찾아내는 것입니다.`

4. 넣을 이미지/자료
   - 좌우 비교 도식.
   - 기존 방식과 공가지도 흐름.

5. 자료를 어디서 구해야 하는지
   - 직접 도형으로 제작한다.

6. 배치 방법
   - 왼쪽: `기존 방식`
     - `등록 빈집 조회 → 현황 확인`
   - 오른쪽: `공가지도`
     - `공공데이터 분석 → 빈집 후보 탐지 → 우선조사 추천`
   - 오른쪽 영역을 블루로 강조.

7. 색상/강조 방법
   - 기존 방식은 회색.
   - 공가지도는 블루/그린.

8. 발표 멘트
   - `공가지도의 핵심은 빈집을 확정 판정하는 것이 아니라, 현장조사 전에 확인해야 할 후보를 먼저 찾아내는 것입니다. 그래서 조회 서비스가 아니라 사전 탐지와 우선조사 추천 서비스로 정의했습니다.`

9. 주의할 점
   - 이 페이지는 문장을 많이 넣지 않는다.
   - `확정 판정이 아니다`를 분명히 말한다.

10. 심사기준 연결
   - 아이디어 명확성.
   - 정책 활용성.

## 5페이지. 솔루션

1. 페이지 제목
   - `공가지도는 빈집 후보를 찾고 우선순위를 제안합니다`

2. 이 페이지의 목적
   - 공가지도의 작동 흐름을 한 번에 보여준다.

3. 넣을 문장
   - `공공데이터 수집 → AI 후보 탐지 → 위험도·활용 가능성 점수화 → 지도 시각화 → 정책 활용 추천`

4. 넣을 이미지/자료
   - 5단계 프로세스 도식.
   - 아이콘 5개.

5. 자료를 어디서 구해야 하는지
   - PowerPoint 아이콘 또는 미리캔버스 기본 아이콘.
   - 아이콘은 선형 스타일로 통일.

6. 배치 방법
   - 가로 플로우 차트.
   - 각 단계 아래 1줄 설명.
   - 마지막 단계에 `정책 의사결정 보조` 표시.

7. 색상/강조 방법
   - 첫 단계는 회색.
   - AI 후보 탐지는 블루.
   - 정책 활용은 그린.

8. 발표 멘트
   - `여러 공공데이터를 교차 분석해 빈집 가능성이 높은 후보를 추정하고, 위험도와 활용 가능성에 따라 우선순위를 제안합니다. 담당자는 이 결과를 보고 현장조사 전에 어디를 먼저 확인할지 판단할 수 있습니다.`

9. 주의할 점
   - `AI가 답을 낸다`보다 `판단 근거를 정리한다`는 표현을 쓴다.

10. 심사기준 연결
   - 데이터 융합.
   - AI 활용.
   - 구현 가능성.

## 6페이지. 서비스 화면

1. 페이지 제목
   - `지도에서 후보와 판단 근거를 한 번에 확인합니다`

2. 이 페이지의 목적
   - 실제 구현된 화면을 보여주고 신뢰도를 높인다.

3. 넣을 문장
   - `지도에서 빈집 후보 위치를 확인하고, 후보 상세 패널에서 AI 추정 근거와 우선조사 정보를 함께 확인합니다.`

4. 넣을 이미지/자료
   - 실제 배포 사이트 캡처.
   - 지도 화면.
   - 후보 상세 패널.
   - 점수/위험도/활용 가능성 표시.

5. 자료를 어디서 구해야 하는지
   - 배포 사이트 또는 로컬 실행 화면.
   - `/map`, `/house/[id]`, `/dashboard` 화면 캡처.

6. 배치 방법
   - 왼쪽 또는 중앙에 서비스 화면을 크게 배치.
   - 오른쪽에 설명 카드 3개:
     - `위치 확인`
     - `판단 근거`
     - `우선조사 추천`

7. 색상/강조 방법
   - 캡처 화면은 원본 그대로.
   - 설명 카드의 아이콘만 블루/그린 사용.

8. 발표 멘트
   - `담당자는 지도에서 후보 위치를 확인하고, 어떤 데이터 근거로 후보가 추천되었는지 함께 볼 수 있습니다. 이 화면은 실제로 구현된 서비스 화면입니다.`

9. 주의할 점
   - 목업처럼 보이지 않게 실제 브라우저 캡처를 사용한다.
   - 개인정보가 보이면 가린다.

10. 심사기준 연결
   - 실제 구현 완성도.
   - 사용성.

## 7페이지. 데이터 활용 구조

1. 페이지 제목
   - `여러 공공데이터를 교차 분석합니다`

2. 이 페이지의 목적
   - 공공데이터 활용이 단순 조회가 아니라 융합 분석임을 보여준다.

3. 넣을 문장
   - 입력: `건축물대장, 에너지 사용량, 위성영상, 인구·상권·교통 데이터, 정책·정비 데이터`
   - 출력: `빈집 후보 점수, 위험도, 활용 가능성, 우선조사 추천, 정책 활용 추천`

4. 넣을 이미지/자료
   - 입력 데이터 5개와 출력 5개를 연결한 아키텍처 도식.

5. 자료를 어디서 구해야 하는지
   - 직접 제작.
   - 공공데이터 이름은 실제 활용 가능 데이터 위주로 쓰고, 아직 연동 전이면 `연동 예정`을 표시.

6. 배치 방법
   - 왼쪽 입력 데이터.
   - 중앙 분석 엔진.
   - 오른쪽 행정 활용 출력.

7. 색상/강조 방법
   - 입력은 회색/블루.
   - 분석은 네이비.
   - 출력은 그린.

8. 발표 멘트
   - `단일 데이터가 아니라 여러 데이터를 교차 분석해 후보를 추정하기 때문에, 현장조사 전 사전 스크리닝 도구로 활용할 수 있습니다. 실제 실증 단계에서는 지자체 빈집대장과 현장 확인 결과로 검증할 계획입니다.`

9. 주의할 점
   - 아직 실제 API 연동이 아닌 데이터는 `개념 설계` 또는 `연동 예정`으로 정확히 표시한다.

10. 심사기준 연결
   - 공공데이터 융합성.
   - 기술 구현 방향.

## 8페이지. 경쟁 서비스 대비 차별성

1. 페이지 제목
   - `기존 플랫폼과 공가지도는 역할이 다릅니다`

2. 이 페이지의 목적
   - 경쟁이 아니라 역할 차이를 명확히 한다.

3. 넣을 문장
   - 표 구성:
     - `구분`
     - `기존 빈집 플랫폼`
     - `공가지도`
   - 행:
     - 핵심 역할: `등록 빈집 조회` / `미등록 빈집 후보 탐지`
     - 데이터 활용: `확인된 정보 중심` / `공공데이터 + AI 분석`
     - 행정 활용: `현황 확인` / `조사 우선순위 추천`
     - 정책 연결: `제한적` / `정비·매입·청년주거 연계`

4. 넣을 이미지/자료
   - 기존 플랫폼 캡처 1개.
   - 공가지도 캡처 1개.

5. 자료를 어디서 구해야 하는지
   - 기존 플랫폼은 공개 서비스 화면 캡처.
   - 공가지도는 직접 캡처.

6. 배치 방법
   - 상단 제목.
   - 중앙 비교표.
   - 하단 한 줄 결론.

7. 색상/강조 방법
   - 기존 플랫폼 열은 회색.
   - 공가지도 열은 연한 블루 배경.

8. 발표 멘트
   - `기존 플랫폼은 공식 등록 정보를 확인하는 데 강점이 있고, 공가지도는 등록되기 전 후보를 찾아내는 데 집중합니다. 그래서 서로 대체 관계라기보다 행정 의사결정의 앞단을 보완하는 역할입니다.`

9. 주의할 점
   - 기존 플랫폼을 부정하지 않는다.
   - `공식 판정`이라는 표현을 쓰지 않는다.

10. 심사기준 연결
   - 차별성.
   - 서비스 필요성.

## 9페이지. 정책 활용 시나리오

1. 페이지 제목
   - `지자체는 조사 전 우선순위를 판단할 수 있습니다`

2. 이 페이지의 목적
   - 실제 행정에서 어떻게 쓰일지 보여준다.

3. 넣을 문장
   - `후보 탐지 → 우선조사 대상 선정 → 현장 확인 → 정비·매입 검토 → 활용 연계`

4. 넣을 이미지/자료
   - 행정 프로세스 흐름도.
   - 담당 부서 또는 정책 활용 카드.

5. 자료를 어디서 구해야 하는지
   - 직접 제작.
   - 지자체 빈집 정비사업 자료를 참고하되 세부 정책명은 검증된 것만 사용.

6. 배치 방법
   - 가로 프로세스.
   - 각 단계 아래 담당자 관점의 행동을 1줄씩.

7. 색상/강조 방법
   - 조사 전 단계는 블루.
   - 현장 확인은 오렌지.
   - 활용 연계는 그린.

8. 발표 멘트
   - `공가지도는 지자체의 빈집 실태조사, 정비사업, 매입 검토, 청년주거·창업공간 연계까지 이어질 수 있습니다. 핵심은 현장조사를 대체하는 것이 아니라, 조사 전에 우선순위를 정하는 것입니다.`

9. 주의할 점
   - 정책 집행 결과를 확정적으로 말하지 않는다.
   - `무조건 절감` 같은 표현 금지.

10. 심사기준 연결
   - 정책 활용성.
   - 현장 적용 가능성.

## 10페이지. 실행 계획과 확장성

1. 페이지 제목
   - `지자체 실증에서 유휴공간 데이터 플랫폼으로 확장합니다`

2. 이 페이지의 목적
   - 프로토타입 이후 로드맵을 현실적으로 제시한다.

3. 넣을 문장
   - `프로토타입 구현 → 공공데이터 추가 연동 → 지역 단위 실증 → 지자체 SaaS → 유휴공간 전체 확장`
   - 하단 강조: `개발자 출신 대표가 직접 구현한 서비스`

4. 넣을 이미지/자료
   - 로드맵 타임라인.
   - 현재 구현 화면 작은 캡처.

5. 자료를 어디서 구해야 하는지
   - 직접 제작.

6. 배치 방법
   - 중앙: 5단계 타임라인.
   - 하단: 실행 역량 강조 박스.

7. 색상/강조 방법
   - 현재 단계는 블루.
   - 다음 단계는 회색.
   - 확장 단계는 그린.

8. 발표 멘트
   - `공가지도는 아이디어 단계가 아니라 직접 구현된 프로토타입을 기반으로, 지자체 실증과 유휴공간 데이터 플랫폼으로 확장할 수 있습니다. 먼저 특정 지역에서 검증하고, 이후 공공데이터 연동과 SaaS 형태로 넓혀가겠습니다.`

9. 주의할 점
   - 매출, 도입 기관, 성과 수치를 임의로 넣지 않는다.

10. 심사기준 연결
   - 실행 가능성.
   - 지속 가능성.
   - 확장성.

## 11페이지. 마무리

1. 페이지 제목
   - `방치되는 공간을 다시 도시의 가능성으로 연결하겠습니다`

2. 이 페이지의 목적
   - 서비스의 비전과 다음 행동을 짧게 남긴다.

3. 넣을 문장
   - `공가지도는 빈집을 나중에 확인하는 지도가 아니라, 먼저 발견하고 다시 연결하는 AI 도시 데이터 플랫폼입니다.`
   - 마지막 문장: `불필요하게 방치되는 공간 0%를 목표로 합니다.`

4. 넣을 이미지/자료
   - QR코드.
   - 서비스 URL.
   - 이메일.
   - 로고 또는 워드마크.

5. 자료를 어디서 구해야 하는지
   - QR코드: 미리캔버스 또는 신뢰 가능한 QR 생성 도구.
   - 서비스 URL: 최종 배포 URL.
   - 이메일: 발표자 이메일.

6. 배치 방법
   - 중앙 큰 문장.
   - 하단 오른쪽 QR코드.
   - 하단 왼쪽 URL/이메일.

7. 색상/강조 방법
   - 제목 네이비.
   - 마지막 문장 중 `도시의 가능성`만 그린 강조.

8. 발표 멘트
   - `공가지도는 방치된 공간을 도시의 문제로만 보지 않고, 다시 활용 가능한 도시 자산으로 전환하는 플랫폼이 되고자 합니다. 오늘 보여드린 프로토타입을 바탕으로 지자체 실증까지 이어가겠습니다. 감사합니다.`

9. 주의할 점
   - `0%`는 비전 문장으로만 사용하고 실적 수치처럼 보이게 하지 않는다.
   - QR코드는 발표 전 반드시 스캔 테스트한다.

10. 심사기준 연결
   - 비전.
   - 실현 의지.
   - 후속 검토 가능성.
