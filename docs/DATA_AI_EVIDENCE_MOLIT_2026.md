# 공가지도 공공데이터/AI 활용 증거 감사

작성일: 2026-06-21
판정 기준: 레포 내부 파일, 실제 코드 검색, 로컬 실행 결과, 공개 출처 확인
최종 업데이트: 2026-06-21 실서비스 전환 작업 반영

## 0. 실서비스 전환 후 핵심 변경

- 실제 공공데이터 연동 추가: `/api/public-data/yeongju`가 `data.go.kr`의 `경상북도 영주시_빈집 현황` 공개 CSV 파일을 직접 수집한다. 현재 검증 결과 `950건`, 기준일 `2025-07-10`, 수집 방식 `data_go_kr_file_download`.
- 공공데이터 API 확장 경로 추가: `DATA_GO_KR_SERVICE_KEY`가 있으면 ODCloud JSON API를 우선 사용하고, 없으면 공개 CSV로 fallback한다.
- 실제 AI API 경로 추가: `/api/ai/analyze`가 서버에서 OpenAI Responses API를 호출하도록 구현되었다. 현재 로컬 환경은 `OPENAI_API_KEY`가 없어 `local_policy_scoring` fallback이 동작한다.
- UI 증거 추가: 홈/대시보드의 `Production data` 패널이 공공데이터 수집 상태, 950건, 기준일, API 키 상태, 우선 조사 후보군을 표시한다.
- 보안 원칙: API 키 값은 서버 전용 환경변수로만 읽고, 클라이언트에는 설정 여부만 표시한다.

## 1. 핵심 판정

현재 레포 기준으로 공공데이터 원천 파일 수집과 OpenAI 연동용 서버 API가 실제 구현되었다. 다만 현 실행 환경에는 `OPENAI_API_KEY`, `DATA_GO_KR_SERVICE_KEY`, `VWORLD_API_KEY`가 없어 OpenAI는 로컬 정책 산식 fallback으로, 공공데이터는 공개 CSV 다운로드 방식으로 동작한다. 발표에서는 "공공데이터 직접 수집은 현재 동작, OpenAI 분석은 키 설정 시 동작하며 현재 환경에서는 fallback"이라고 정확히 말해야 한다.
